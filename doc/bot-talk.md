# ==================================================
# KennyBot Discord LLM Agent - Full Spec for Codex
# ==================================================

# =========================
# 概要
# =========================
このBotはDiscord上で動作するLLMエージェントであり、
ユーザーの発話に対して自然な会話を行いながら、
必要に応じて検索・ツール実行・履歴参照を行う。

特徴:
- LLM主導のPlanning
- ツール選択型アーキテクチャ
- グローバルQueueによる同期実行
- 会話型一次応答 + 最終回答
- JSONプラン埋め込み方式
- デバッグモードで全内部可視化

# =========================
# 出力フォーマット（必須）
# =========================

LLMは必ず以下の形式で出力すること:

<USER_MESSAGE>
ユーザーに見せる自然な会話文
</USER_MESSAGE>

<SYSTEM_PLAN>
{
  "intent": "...",
  "selected_model": "...",
  "needs_web_search": true/false,
  "needs_rag": true/false,
  "needs_additional_history": true/false,
  "queries": [],
  "tools_to_call": [],
  "response_mode": "...",
  "confidence": 0.0
}
</SYSTEM_PLAN>

※Discordには USER_MESSAGE のみ送信する
※SYSTEM_PLANは内部処理専用

# =========================
# 初期コンテキスト（必須）
# =========================

LLMには必ず以下を渡す:

- system prompt
- user_text
- now_datetime
- timezone
- user_id
- channel_id
- guild_id
- reply_to_message_id
- attachment metadata
- available_tools
- ユーザーの直近履歴3件（最低）

# =========================
# 履歴ルール
# =========================

- 最初に必ず3件渡す
- LLMが必要と判断した場合のみ追加取得
- 上限:
  - 初期: 3件
  - 追加: 最大20件

# =========================
# フロー
# =========================

[1] メッセージ受信
↓
[2] 最小コンテキスト構築
↓
[3] LLM Planning
↓
[4] Queue投入
↓
[5] Worker実行（1件ずつ）
↓
[6] ツール実行
↓
[7] LLM Final Answer
↓
[8] Discord送信 / 編集

# =========================
# Queue仕様
# =========================

- グローバルQueue（全サーバー共通）
- 実行は常に1件のみ
- FIFO

Request構造:

{
  "request_id": "...",
  "user_id": "...",
  "channel_id": "...",
  "guild_id": "...",
  "user_text": "...",
  "plan": {},
  "status": "QUEUED | RUNNING | DONE"
}

# =========================
# 表示ルール
# =========================

■ 待機あり
<@USER_ID> Kennybot取り込み中... 現在 n 人待ち

■ 待機なし
→ いきなり自然な一次応答

■ 推論中
<@USER_ID> gemini-2.5-flashが4秒推論中

■ 最終回答
自然な会話 + 質問で終了

# =========================
# 一次応答ルール（重要）
# =========================

必ず以下を含める:

1. 軽い受け止め
2. 意図の解釈
3. 検索方針説明（会話で）
4. 待機メッセージ

例:
「今日の天気を知りたい感じだね。
今日ベースで気温と降水確率を見てくるよ」

※検索語は出さない

# =========================
# 最終回答ルール
# =========================

必ず以下の構造:

1. 結論
2. 補足
3. 軽いアドバイス
4. 質問（1つ）

例:
「今日は晴れで23℃くらい。
夕方だけ雨の可能性あり。
傘あると安心かも。
今日は出かける予定？」

# =========================
# 検索仕様
# =========================

- LLMが検索語生成
- 複数クエリOK
- 上限: 3件

例:
"queries": [
  "関東 天気 今日",
  "関東 降水確率",
  "weather kanto today"
]

# =========================
# ツール一覧
# =========================

- search_web

- get_user_info

- analyze_attachment

# =========================
# エラー処理
# =========================

必ずprogress messageを終端させる:

「処理中にエラーが発生しました」

# =========================
# 禁止事項
# =========================

- SYSTEM_PLANをユーザーに見せる
- 無機質な応答のみ
- 会話なしの結果返却
- 検索語の直接表示

# =========================
# ゴール
# =========================

このBotは

「自然な会話をしながら裏で検索・推論するエージェント」

である。

単なるChatbotではなく、
「会話 + 推論 + ツール実行」を統合したものとする。
